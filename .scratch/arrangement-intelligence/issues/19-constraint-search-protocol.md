# Serializable constraint and search protocol

Status: ready-for-agent

Type: AFK

User stories: U4, U6

## What to build

Introduce reusable constraint, evaluator, Search Adapter, and honest Search Outcome contracts without one universal musical state.

## Acceptance criteria

- [ ] Constraints are serializable, versioned, scoped, and provenance-bearing.
- [ ] Evaluator evidence separates completeness, basis, authority, and presentation.
- [ ] Search distinguishes candidate found, exhaustion, exhaustive infeasibility, uncertainty, cancellation, and failure.
- [ ] Checkpoints and completeness certificates retain exact execution identity.
