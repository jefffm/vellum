# Versioned Evaluation Harness core

Status: ready-for-agent

Type: AFK

User stories: U6, U9

## What to build

Create the persistent Evaluation Harness spine with isolated end-to-end and component cases and immutable resolved execution manifests.

## Acceptance criteria

- [ ] End-to-end cases begin from source and briefs; component cases pin reviewed inputs.
- [ ] Resolved manifests pin all transitive case, expectation, evaluator, mutation, profile, fixture, and protocol digests.
- [ ] Absolute results separate applicability, execution, outcome, evidence, authority, and presentation.
- [ ] A machine-readable command produces an Evaluation Run and Card for the first loop.
