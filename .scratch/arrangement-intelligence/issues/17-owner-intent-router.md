# Canonical Owner-intent router

Status: ready-for-agent

Type: AFK

User stories: U2, U3, U5

## What to build

Classify every score-anchored request before mutation so source corrections, plan revisions, score edits, interpretations, commitments, and exceptions use their correct canonical layer.

## Acceptance criteria

- [ ] The UI shows proposed canonical layer, scope, consequence, and confirmation requirement.
- [ ] Source and Analysis corrections follow their existing lineage rather than becoming arrangement constraints.
- [ ] Low-consequence explanations require no unnecessary confirmation.
- [ ] Ambiguous consequential requests remain uncommitted until resolved.
