# Owner Knowledge and Personal Defaults workbench

Status: complete

Type: AFK

## What to build

Expose the local reviewed knowledge and preference system so the Owner can inspect provenance, approve Knowledge Candidates and Personal Default candidates, correct or reject them, see where they affected an arrangement, and release them. Detailed evidence remains available without burdening the default arrangement flow.

## Acceptance criteria

- [x] Reviewed Historical Practice Claims show source, scope, confidence, and current status.
- [x] Knowledge and default candidates can be approved, corrected, rejected, or released.
- [x] Arrangement explanations identify applied Personal Defaults and their precedence.
- [x] Selection Context can propose a scoped default without silently learning it.
- [x] Released defaults stop affecting future generation while prior arrangements remain reproducible.
